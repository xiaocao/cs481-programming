CS-481: Tokenizer documentation summary

The supplied Java utilities, in packages nlp.token and nlp.diff, supply several useful classes. This document briefly explains the Tokenizer and Token classes, which are used to read in and tokenize an input stream. You can also use Tokens in your internal representation of the text and its tags. See nlp/docs for more information. 
public class Tokenizer
extends java.lang.Object

Tokenize an input stream.

Typical use: 

 Tokenizer t = new Tokenizer(reader,false);
 Token tok;
 while((tok = t.getNext()) != null)
 {
     .....
 }
The idea is to create a Tokenizer on an input reader and then to use it to read the tokens from the file. Words and numbers are single tokens each, as are individual punctuation marks. The second option to the constructor specifies if slash-separated parts-of-speech should be read in as the "POS" attributes of the individual tokens (see Token below and the Gene.txt data file. 

public Tokenizer(java.io.Reader r,
                 boolean getPOS)
Makes a tokenizer that reads from the given reader

Parameters:
r - Where to read from
getPOS - Whether the tokenizer should read in parts-of-speech for each token, separated by a slash, as in:
the/DT0 books/NN2 are/VBZ on/PRP the/DT0 table/NN1 ./PUN

public class Token
extends java.lang.Object

A class for storing a token. It contains a String (the token's "name") and a set of associated attributes (named by Strings with Object values).

Typical use: 

 Tokenizer tknz = new Tokenizer(reader,true);
 Token  tok = tknz.getNext();
 String name   = tok.getName();
 String pos = tok.getAttrib("POS");
 tok.putAttrib("PERS-TAG","B");
